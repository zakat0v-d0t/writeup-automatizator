# writeup-automatizator

Удобная CLI-утилита для автоматической генерации красивых и структурированных райтапов после решения CTF-тасков.

## Требования 

- Python 3.13+

## Установка 

```
# 1. Клонируйте репозиторий  
git clone https://github.com/zakat0v-d0t/writeup-automatizator.git  
cd writeup-automatizator  
  
# 2. Установите зависимости и саму утилиту  
uv venv
source .venv/bin/activate
uv pip install -e .
```

## Использование

После установки утилита будет доступна глобально в вашем виртуальном окружении под командами `writeup` и `wup`.

### Создание нового райтапа

Запустите команду инициализации:

```bash
wup init
```

Вы также можете явно указать директорию для сохранения, чтобы все райтапы складывались в единую базу знаний:
```bash
wup init --output-dir ~/my_ctf_writeups
```

**Процесс:**

1. Утилита запросит базовую информацию: название таска, категорию (pwn, web, crypto и т.д.), сложность и флаг.

2. Далее начнется сбор шагов решения. Для каждого шага можно указать заголовок, текстовое описание и использованные команды (поддерживается перенос строк через `\\n`). Язык кода для подсветки синтаксиса определяется автоматически (Python, C, SQL, JS, PHP, JSON, ASM, Bash).

3. Чтобы завершить ввод шагов, просто оставьте заголовок очередного шага пустым и нажмите `Enter`.

4. В консоль будет выведен путь к сохраненному `.md` файлу.

### Редактирование существующего райтапа

Команда `edit` позволяет открыть уже существующий файл состояния (`.json`) или готовый райтап (`.md`) в вашем системном редакторе:

```bash
wup edit <путь_к_файлу>
```

Вы также можете явно указать редактор с помощью флага `--editor` или `-e`:

```bash
wup edit <путь_к_файлу> -e nano
```

Если вы отредактировали `.json` файл, вам потребуется перегенерировать `.md` файл, чтобы изменения вступили в силу.

### Генерация Markdown из JSON

Команда `generate` используется для создания или обновления `.md` файла на основе состояния из `.json` файла:

```bash
wup generate <путь_к_json>
```

Она пересоздаст Markdown-документ, используя актуальные данные и шаблон.

### Просмотр списка райтапов

Команда `list` сканирует папку `writeups` и выводит список всех найденных тасков в виде удобной таблицы:

```bash
wup list
```
Можно также передать кастомную директорию через флаг `--dir`.

### Публикация в Telegraph

Команда `publish` публикует райтап в Telegraph. Она автоматически конвертирует Markdown в HTML и возвращает готовую ссылку.

```bash
wup publish <путь_к_json>
```

По умолчанию создается новый анонимный аккаунт. Если хочешь публиковать от своего имени (никнейма), сначала создай и привяжи аккаунт командой:

```bash
wup auth-telegraph "твой_никнейм"
```

*Прим.: Токен безопасно сохраняется в файл `~/.writeup.env` с правами доступа только для владельца (600).*

### Автоперевод райтапа

Команда `translate` переводит текст райтапа на другой язык (код остается нетронутым) с помощью бесплатного Google Translate:

```bash
wup translate <путь_к_json> --target-lang <код_языка>
```

Например, для перевода на английский (по умолчанию):
```bash
wup translate writeups/2026/web/task.json --target-lang en
```
Сохранится как `task_en.json`. Затем используйте `wup generate` для создания `.md`.

### Просмотр всех доступных команд

```bash
wup --help
```

```

- **Флаг:** `ctf{example_flag}`  
- **Команда:** r007s  
- **Дата:** 2026-07-01  
  
## Описание  
Простейшее переполнение буфера...  
  
## Решение  
  
### Шаг 1: Разведка  
Проверил бинарник утилитой checksec.  
  
```bash  
checksec ./binary
```
